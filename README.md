# LearnAnything — LLM-powered Duolingo-style learning platform

Type any topic. Get a full course (units → lessons → exercises) generated by an LLM.
Play through Duolingo-style lessons with XP, streaks, hearts, and a weekly leaderboard.

## Stack
- **Frontend**: React 18 + Vite + TypeScript + Tailwind + TanStack Query + React Router + Zustand + Framer Motion
- **Backend**: FastAPI + SQLAlchemy 2.0 (async) + Alembic + Pydantic v2 + minimal username/password JWT auth (bcrypt + PyJWT)
- **DB**: Postgres 16
- **LLM**: OpenAI (`gpt-4o`) via JSON-mode responses
- **Deploy**: Docker Compose → Coolify on Hostinger

## 1. Local development

```bash
cp .env.example .env
# Edit .env — at minimum set OPENAI_API_KEY and SECRET_KEY.
# Generate a secret: python -c "import secrets; print(secrets.token_urlsafe(64))"

docker compose up --build
```

- Frontend: http://localhost:5173
- API docs: http://localhost:8000/docs
- Postgres: `localhost:5432` (user/pass from `.env`)

The backend container runs `alembic upgrade head` on every start, so the schema is always current.

### First-time migration
The repo ships without a baseline migration. Generate one once after first `docker compose up`:

```bash
docker compose exec backend alembic revision --autogenerate -m "initial schema"
docker compose exec backend alembic upgrade head
```

Commit the generated file under `backend/alembic/versions/`.

### Running without Docker
```bash
# Backend
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
alembic upgrade head
uvicorn app.main:app --reload

# Frontend (in another terminal)
cd frontend
npm install
npm run dev
```

## 2. Deploying on Coolify (Hostinger)

1. **Push this repo** to a git provider Coolify can read.
2. In Coolify, create a new **Docker Compose** resource pointing at the repo and select `docker-compose.coolify.yml`.
3. Add these environment variables in Coolify's UI:
   - `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`
   - `SECRET_KEY` (generate with `python -c "import secrets; print(secrets.token_urlsafe(64))"`)
   - `OPENAI_API_KEY` (and optionally `OPENAI_MODEL`, defaults to `gpt-4o`)
   - `DOMAIN` (e.g. `learn.example.com`) and `VITE_API_URL` (e.g. `https://api.learn.example.com`)
   - `DEMO_USERNAME`, `DEMO_PASSWORD` (optional — defaults to `demo` / `learnnow`)
4. In Coolify's **Domains** panel:
   - Route `${DOMAIN}` → `frontend` service, port `80`
   - Route `api.${DOMAIN}` → `backend` service, port `8000`
5. Click **Deploy**. Coolify's Traefik provisions HTTPS via Let's Encrypt automatically.

Migrations run on every deploy (the backend container's CMD runs `alembic upgrade head` before starting Uvicorn).

### Backups
Coolify can schedule automatic backups of the `postgres_data` volume — enable in the resource's **Storage** tab.

## 3. Architecture notes

- **Generation is hybrid**: course outlines (units + lesson stubs) are generated synchronously on course creation in a background task. Lesson exercises are generated lazily on first lesson load and cached forever. This keeps the upfront wait short while the long tail of lessons stays cheap.
- **Grading is server-side**: the `/lessons/{id}` endpoint strips answer fields from the exercise payload before sending it to the client. Submissions are graded in `app/services/grading.py`.
- **XP is an append-only ledger** (`xp_event` table). Total XP = `SUM(amount)`. Makes leaderboards trivial and gives free analytics.
- **Streak logic** lives in `gamification_service.extend_streak_if_eligible` — yesterday extends, today is a no-op, anything older resets to 1.
